This implementation guide is an Australian realm implementation guide of the HL7TM FHIR® specification to represent atomic Medicare records in the My Health Record system and related applications.

This implementation guide is based on FHIR® version 3.0.1.

Scope

Use Cases

The following is the expected scenario:

  • DHS Medicare sends Medicare records to the My Health Record system

Profiles

The FHIR® profiles that form part of this implementation guide are shown below. The profiles defined in this implementation guide do not include profile-specific mappings to another format. The base FHIR® STU3 mapping content for each of the resources referenced in this implementation guide can be found on the applicable resource documentation in the FHIR Release 3 (STU).

Profiles that define representation of Pharmaceutical Benefits Schedule (PBS) claim items or Medicare Benefits Schedule (MBS) claim items:

  • Explanation of Benefit Medicare
  • Medication Request Pharmaceutical Benefits Scheme
  • Medication Pharmaceutical Benefits Scheme
  • Referral Request Medicare Benefits Schedule Department of Veterans’ Affairs

Profiles that define representation of donation decision information from the Australian Organ Donor Register:

  • Consent Australian Organ Donor Register
  • BodySite Australian Organ Donor Register

Profile that defines the representation of vaccination information from the Australian Immunisation Register:

  • Immunization Australian Immunisation Register
