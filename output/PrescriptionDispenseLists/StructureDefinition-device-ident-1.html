Device with Mandatory Identifier [Draft]

Usage scenarios

Implementation guidance

SD.9.1 Formal Views of Profile Content

The official URL for this profile is: 

http://ns.electronichealth.net.au/ci/fhir/3.0/StructureDefinition/device-ident-1

The purpose of this profile is to define a representation of a device in an Australian context where an identifier is mandatory.

Must Support

In the context of this profile Must Support SHALL be interpreted as follows:

  • The system SHALL be able to include the element when authoring a resource where data is available

  • The system SHALL be able to store and retrieve the element

  • The system SHALL be able to process resources containing the element

This profile builds on AUBaseDevice.

This profile was published on Tue Mar 17 04:12:25 AEST 2020 as a draft by Australian Digital Health Agency.
