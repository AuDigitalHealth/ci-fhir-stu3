SD.10 StructureDefinition-observation-norelevantfinding-1

Assertion of No Relevant Finding [Draft]

Implementation guidance

For the expected usage scenarios in this implementation guide:

  • It is expected that status will be ‘final’.

SD.10.1 Formal Views of Profile Content

The official URL for this profile is: 

http://ns.electronichealth.net.au/ci/fhir/3.0/StructureDefinition/observation-norelevantfinding-1

The purpose of this profile is to provide a statement that there is a clinical judgement that there are no items of specific interest, e.g. no allergies, no medications.

Must Support

In the context of this profile Must Support SHALL be interpreted as follows:

  • The system SHALL be able to include the element when authoring a resource where data is available

  • The system SHALL be able to store and retrieve the element

  • The system SHALL be able to process resources containing the element

This profile builds on AUAssertionNoRelevantFinding.

This profile was published on Mon Nov 04 14:31:14 AEST 2019 as a draft by Australian Digital Health Agency.
